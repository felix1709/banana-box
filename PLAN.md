# Banana Box — 桌面提示词库设计方案

> 状态：已确认，待实现
> 创建日期：2026-06-27
> 类型：Tauri 2 桌面应用（Rust 壳 + Vue 3 前端）

## 1. 项目概述

Banana Box 是一个**轻量级桌面提示词库**，常驻系统托盘，随时点击或按快捷键唤出悬浮面板，点击提示词即复制到剪贴板。支持分类整理、关键词检索、参考图预览，数据本地存储可一键导出/导入，方便换电脑迁移。

### 核心需求

| # | 需求 | 实现要点 |
|---|------|---------|
| 1 | 悬浮在桌面随时点击随时使用 | 系统托盘常驻 + 左键弹出悬浮面板 + 全局快捷键 |
| 2 | 存储提示词随时点击复制 | 卡片点击即复制 content 到剪贴板 + toast 反馈 |
| 3 | 保存本地随时切换电脑无缝衔接 | 本地数据文件夹 + 一键导出/导入 `.zip` |
| 4 | 提示词有参考图可自由设置预览 | 每条提示词可附加参考图，缩略图 + 点击放大 |
| 5 | 整体轻量级小工具 | Tauri 2 壳，安装包 ~5MB，内存 <80MB |
| 6 | 分类整理 + 关键词快速检索 | 分类树筛选 + 搜索框实时匹配 |

## 2. 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 桌面壳 | Tauri 2（Rust） | 真正轻量，安装包几 MB；需一次性安装 Rust 工具链 |
| 前端框架 | Vue 3 + TypeScript | 模板直观、中文资料丰富、对初学者友好 |
| 构建工具 | Vite | Tauri 官方推荐，热更新快 |
| 状态管理 | Pinia | Vue 3 官方推荐，API 简洁 |
| UI 组件库 | Naive UI | Vue 3、按需引入（Tree-shaking）不增体积，适合轻量工具 |
| 测试 | Vitest（前端）+ Rust 单元测试（后端） | 逻辑层先行 |

## 3. 整体架构

职责清晰两层，前后端通过 Tauri **命令（IPC）** 通信：

```
┌──────────────────────────────────────────┐
│  前端（Vue 3 + TS，WebView 渲染进程）       │
│  ┌──────────┬──────────┬───────────────┐ │
│  │ 列表/搜索 │ 分类树   │ 编辑/设置      │ │
│  └─────┬────┴─────┬────┴───────┬───────┘ │
│        │  Pinia store（数据/状态）        │ │
└────────┼──────────┼─────────────┼────────┘
         │  Tauri IPC 命令（invoke）        │
┌────────▼──────────▼─────────────▼────────┐
│  后端（Rust，主进程）                       │
│  文件读写 / 剪贴板 / 托盘 / 全局快捷键 / 窗口 │
└──────────────────────────────────────────┘
              │
┌─────────────▼──────────────┐
│  本地数据文件夹               │
│  %APPDATA%/banana-box/      │
│  ├─ library.json            │
│  └─ images/                 │
└─────────────────────────────┘
```

**原则：** 前端只管界面和状态，所有系统操作（文件、剪贴板、托盘、快捷键）走 Rust 后端命令。这样前端可在纯浏览器里开发调试，逻辑也便于测试。

## 4. 数据模型

数据文件 `library.json`，存在本地数据文件夹。版本号用于未来迁移。

```json
{
  "version": 1,
  "categories": [
    { "id": "uuid", "name": "写作", "color": "#f59e0b", "order": 0 }
  ],
  "prompts": [
    {
      "id": "uuid",
      "title": "总结助手",
      "content": "请把以下内容总结成三点...",
      "categoryId": "uuid",
      "tags": ["总结", "中文"],
      "image": "images/abc123.png",
      "createdAt": 1719000000,
      "updatedAt": 1719000000
    }
  ],
  "settings": {
    "hotkey": "Ctrl+Shift+B",
    "theme": "auto"
  }
}
```

**参考图存储：** 上传时把图片**复制到 `images/` 文件夹**，库里只存相对路径（如 `images/abc123.png`）。不嵌 base64，文件体积小、导出/导入简单。删除提示词时同步清理对应图片。

**ID 生成：** 使用 UUID v4，避免合并冲突。

**时间戳：** Unix 秒级时间戳。

## 5. 界面布局

悬浮面板：点托盘图标或按快捷键后，在鼠标附近弹出，点击面板外部自动收起。

```
┌─────────────────────────────────────┐
│ 🔍 [搜索提示词、标签...]      ⚙️ ＋  │  顶栏：搜索 + 设置 + 新建
├──────┬──────────────────────────────┤
│ 全部  │ ┌──────────────────────────┐│
│ 写作  │ │ 📌 总结助手        [图]📋 ││  卡片：标题 + 缩略图 + 复制
│ 翻译  │ │ 请把以下内容总结成三点... ││
│ 编程  │ └──────────────────────────┘│
│ ＋新建│ ┌──────────────────────────┐│
│      │ │ 📌 翻译润色        [图]📋 ││
│      │ │ 把下面这段翻译成地道英文...││
│      │ └──────────────────────────┘│
└──────┴──────────────────────────────┘
```

- **顶栏：** 搜索框（实时过滤 title/content/tags）、设置按钮、新建按钮
- **左栏分类树：** 含"全部"项，可增删改、拖拽排序；点击筛选右栏
- **右栏卡片列表：** 当前分类下的提示词；点击卡片任意位置 = 复制 content；缩略图点击放大查看
- **编辑弹窗：** 新建/编辑提示词时弹出，填标题、内容、分类、标签、上传参考图
- **设置弹窗：** 全局快捷键配置、主题、导出/导入

面板默认尺寸约 720×520，可调，关闭时记忆位置/尺寸。

## 6. 核心功能流程

### 6.1 唤起与收起
- 托盘图标常驻；左键点击 → 面板在鼠标附近弹出
- 全局快捷键 `Ctrl+Shift+B`（可在设置改）→ 切换面板显示/隐藏
- 点击面板外部 → 自动收起（失焦隐藏）
- 托盘右键菜单：显示/退出

### 6.2 复制提示词
- 点击卡片 → 前端调 Rust 命令 `copy_to_clipboard(content)` → 写入系统剪贴板
- 成功后显示 "已复制 ✓" toast（1.5s 自动消失）
- 卡片 hover 时高亮，复制后短暂闪一下确认

### 6.3 检索
- 搜索框输入即时过滤，匹配 title / content / tags（不区分大小写）
- 与左栏分类筛选叠加（在当前分类内搜索）
- 空结果时显示"未找到匹配的提示词"

### 6.4 分类整理
- 左栏分类可增删改；删除分类时其下提示词归入"未分类"
- 拖拽分类排序，写回 `categories[].order`
- 提示词可在编辑弹窗里选分类

### 6.5 参考图
- 编辑弹窗里点"上传图片" → 前端调 Rust 命令 `save_image(fileBytes)` → 存 `images/{uuid}.png`，返回相对路径
- 卡片显示缩略图；点击缩略图弹大图浮层
- 删除提示词 → 同步删除对应图片文件
- 支持常见格式：png/jpg/webp

### 6.6 导出/导入
- **导出：** 设置页点"导出" → Rust 命令 `export_library()` → 把 `library.json + images/` 打成 `banana-box-export-YYYYMMDD.zip`，让用户选保存位置
- **导入：** 设置页点"导入" → 选 `.zip` → Rust 解压 → 校验版本 → 选择"合并"或"覆盖"
  - 合并：按 id 去重，已有 id 跳过或覆盖（提示用户）
  - 覆盖：清空当前库，替换为导入内容（需二次确认）

## 8. 测试与验证策略

- **Rust 后端单元测试：** `library.rs` 的读写、迁移、导出/导入打包解包逻辑
- **前端单元测试（Vitest）：** Pinia `library` store 的增删改查、搜索过滤、分类筛选逻辑（纯逻辑，不依赖 Tauri）
- **手动验收：** `pnpm tauri dev` 跑起应用，逐条验证 6 个核心需求
- **完成前验证：** `pnpm typecheck` + `pnpm lint` + `pnpm test` 全绿，再交付

## 9. 范围与不做（YAGNI）

本期**不做**，保持轻量：
- 不做云同步 / 账号体系（用导出/导入代替）
- 不做 AI 生成提示词（纯存储工具）
- 不做多端同步协作
- 不做提示词分享市场
- 不做富文本编辑（content 纯文本）

未来可扩展但不在本期：Markdown 渲染、暗色主题切换、多语言、变量模板（{{var}} 占位填充）。

## 10. 风险与对策

| 风险 | 对策 |
|------|------|
| Rust 工具链安装对初学者有门槛 | 一次性安装，文档写清步骤；日常开发只写 TS |
| Tauri 2 API 变动 | 锁定 Tauri 2.x 稳定版，参考官方文档 |
| 参考图体积膨胀 | 图片存文件不嵌 base64；未来可加压缩 |
| 导入合并冲突 | 按 id 去重 + 用户选择覆盖策略 + 二次确认 |
| 全局快捷键冲突 | 设置页可改快捷键 |
